## 1. 기술 면접 대비 (Q&A)

### Q. CSS Flexbox 레이아웃 내에서 텍스트 말줄임표(truncate)가 적용된 자식 요소 때문에 레이아웃이 깨지거나 다른 자식 요소가 밀리는 현상이 발생했을 때, 어떻게 해결했나요?

**A. 문제 인식 ➔ 대안 분석 및 Trade-off 평가 ➔ 최종 설계 합리화**

* **문제 인식**:
  상세보기 모달에서 식당 명칭이 길어질 경우, 우측에 위치한 '룰렛 추가/제외' 버튼이 모달 영역 밖으로 밀려나 레이아웃이 깨지는 현상이 발생했습니다. 식당 명칭을 렌더링하는 `<DialogTitle>` 엘리먼트에 `truncate` (`overflow: hidden; text-overflow: ellipsis; white-space: nowrap;`) 클래스를 부여했음에도 텍스트가 잘리지 않고 버튼을 밀어냈습니다. 이는 CSS Flexbox 표준 스펙상 flex item의 `min-width` 기본값이 `auto`(또는 `min-content`)로 지정되어 있어, 텍스트가 줄어들지 않고 고유의 너비를 강제하려 하기 때문입니다.

* **대안 분석 및 Trade-off 평가**:
  * **대안 1: 모달 팝업의 너비(`max-w-md` ➔ `max-w-lg`)만 단순히 키우기**
    * *장점*: 공간이 넓어져 보다 많은 텍스트를 한눈에 볼 수 있습니다.
    * *단점*: 모바일 환경(예: 375px 이하 화면)에서는 팝업의 절대 너비가 결국 뷰포트 너비로 제한되므로 동일한 레이아웃 깨짐 현상이 여전히 발생합니다.
  * **대안 2: flex item인 `<DialogTitle>`에 `min-w-0` 적용**
    * *장점*: 화면 크기와 관계없이 flex item이 최소 콘텐츠 크기 이하로 축소(`min-width: 0`)될 수 있게 하므로 `truncate`가 정상적으로 동작하여 레이아웃이 안전하게 유지됩니다.
    * *단점*: 식당 이름이 극단적으로 길면 뒷부분이 가려져 생략되는 한계가 있습니다.

* **최종 설계 합리화 (이중 대응)**:
  사용자 경험을 극대화하기 위해 **대안 1과 대안 2를 조합한 이중 대응 설계**를 채택했습니다. 
  1. 모달의 최대 너비를 기존 `max-w-md`(448px)에서 `max-w-lg`(512px)로 변경하여 데스크톱 및 태블릿 화면에서 시원하고 쾌적한 뷰를 제공합니다.
  2. 동시에 `<DialogTitle>`에 `min-w-0` 클래스를 부여하여 모바일 기기 또는 극단적으로 긴 텍스트 입력 시에도 레이아웃이 유연하게 축소되면서 말줄임표(`...`) 처리되도록 방어막을 구축했습니다. 이를 통해 어떤 디바이스 환경에서도 깨짐이 발생하지 않는 반응형 UI를 달성했습니다.

## 2. 포트폴리오 성과 기술서 (개조식)

### 📌 핵심 트러블슈팅: 모달창 내 장문 텍스트에 의한 Flexbox 레이아웃 붕괴 해결

* **Issue (문제)**
  * 식당 상세보기 모달 팝업에서 식당 이름이 길어질 경우, 우측의 '룰렛 추가/제외' 기능 버튼이 팝업 너비 밖으로 밀려나 레이아웃이 깨지고 버튼 클릭이 불가능해지는 현상 발견.
  * `truncate` 스타일을 적용했음에도 Flexbox 스펙상 자식 요소의 기본 최소 너비(`min-width: auto`) 제약 때문에 텍스트 영역이 축소되지 않고 영역을 확장하는 문제 확인.

* **Action (해결 조치)**
  * **데스크톱 가독성 개선**: 정보 탐색 편의성을 위해 모달창의 전체 최대 가로 크기를 기존 448px(`max-w-md`)에서 512px(`max-w-lg`)로 확장.
  * **Flexbox 제약 조건 해결**: 텍스트를 렌더링하는 `<DialogTitle>` 컴포넌트에 `min-w-0` (`min-width: 0px`) 속성을 부여하여 flex 컨텍스트 내에서 텍스트의 고유 크기 제약을 해제하고 축소 가능하도록 수정.

* **Result (성과)**
  * 식당 이름의 길이에 관계없이 레이아웃이 팝업 영역 내부에서 유동적으로 조절되며 정상적으로 우측 버튼이 배치됨.
  * 모바일(가로 320px~375px 기기)부터 데스크톱까지 기기 해상도에 맞춰 유연하게 텍스트가 말줄임표(`...`) 처리되고 기능 버튼이 깨짐 없이 동작하는 반응형 견고성 확보.
